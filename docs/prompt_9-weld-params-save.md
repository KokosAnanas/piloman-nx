В файле `libs/welds/ui-weld-params-widget/src/lib/weld-params-widget.component.html` найдите разметку заголовка:

```html
<ng-template pTemplate="header">
  <div class="flex items-center gap-2 px-4 pt-4">
    …
  </div>
</ng-template>
```

В правой части этого `<div>` необходимо добавить кнопку PrimeNG:

- Использовать компонент `<p-button>` с подписью `label="Сохранить изменения"`. Кнопка должна быть видна, но деактивирована, пока пользователь не изменил ни одного поля.
- Разместите кнопку справа от существующих элементов (например, добавьте контейнер с классом `ml-auto` и поместите в него `<p-button>`). Используйте свойство `[disabled]="!hasUnsavedChanges"`, где `hasUnsavedChanges` — булево поле класса компонента, отвечающее за состояние формы.

Необходимо реализовать отслеживание изменений полей формы:

1. В компоненте `WeldParamsWidgetComponent` создайте флаг `hasUnsavedChanges = false` и структуру (например, `changedFields: Set<string> = new Set()`) для хранения имён изменённых полей.
2. Для каждого поля, связанного через `[(ngModel)]`, добавьте обработчик `ngModelChange`:
   ```html
   <p-inputNumber [(ngModel)]="weld.diameter" (ngModelChange)="onFieldChange('diameter')" … />
   ```
   Аналогично для толщин, уровня качества, способа сварки, типа соединения, даты и примечаний. Метод `onFieldChange(field: string)` должен:
  - Устанавливать `hasUnsavedChanges = true`;
  - Добавлять имя поля в `changedFields`.

3. Для визуального обозначения несохранённых изменений:
  - Создайте CSS‑класс (например, `.unsaved`) с жёлтым фоном или рамкой и/или иконкой `pi-exclamation-triangle`.
  - Применяйте этот класс условно: `[ngClass]="{ 'unsaved': changedFields.has('diameter') }"` для соответствующего поля.
  - Можно дополнительно отобразить маленькую иконку «!» рядом с label, чтобы пользователь видел, какие поля требуют сохранения.

4. Добавьте метод `saveChanges()` в `WeldParamsWidgetComponent`:
  - Вызывает сервис для обновления текущего объекта `weld` в базе данных (например, `weldService.updateWeld(this.weld)`).
  - После успешного обновления очищает `changedFields`, сбрасывает `hasUnsavedChanges = false` и убирает подсветку.

5. Свяжите кнопку с методом сохранения:
   ```html
   <p-button label="Сохранить изменения" (onClick)="saveChanges()" [disabled]="!hasUnsavedChanges"></p-button>
   ```

Таким образом:

- После первого изменения любого поля форма отмечается как имеющая несохранённые изменения, кнопка становится активной и изменённые поля подсвечиваются.
- При нажатии «Сохранить изменения» данные отправляются в базу данных, подсветка снимается и кнопка снова блокируется до следующего изменения.
