# Задача: Column Toggle рядом с «Добавить стык» (PrimeNG Table)

Контекст: страница **«Реестр сварных соединений»** (существующая страница с `p-table`). Используем PrimeNG + TailwindCSS, Angular standalone, Nx-архитектура проекта сохраняется (ничего лишнего в apps/web).

## Что сделать
1) В шапке страницы, **перед** кнопкой **«Добавить стык»**, добавь компонент **Column Toggle** для `p-table` по примеру из документации PrimeNG: https://primeng.org/table#column-toggle

2) Column Toggle должен управлять видимостью колонок таблицы (включать/выключать столбцы).

3) **По умолчанию выключены** (скрыты) столбцы с заголовками:
- **«Способ сварки»**
- **«Тип соединения»**
- **«Статус»**
- **«Дата сварки»**

Все остальные столбцы должны быть включены по умолчанию.

## UI требования
- Порядок слева направо: **Column Toggle** → **Добавить стык**
- Column Toggle реализуй через `p-multiSelect` (как в примере PrimeNG) с опциями-колонками.

## Технические требования
1) Не ломай текущую таблицу и её разметку — просто добавь поддержку `cols` и `selectedColumns`:

2) Инициализация:
  - `selectedColumns` = все колонки **кроме** тех четырёх, которые должны быть скрыты по умолчанию.
  - В UI эти четыре должны быть **unchecked** изначально.

3) Таблица должна рендерить колонки только из `selectedColumns`

4) Никаких новых страниц/роутов. Всё в существующем компоненте страницы реестра.

## Acceptance criteria
- Column Toggle виден рядом с кнопкой «Добавить стык», стоит **перед** ней.
- При выключении/включении колонок таблица динамически скрывает/показывает столбцы.
- При первом открытии страницы скрыты: «Способ сварки», «Тип соединения», «Статус», «Дата сварки».
